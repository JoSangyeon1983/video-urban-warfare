# 프롬프트 검수 워크플로 (Codex)

> **이미지 생성 프롬프트를 ChatGPT에 넣기 전에 Codex(GPT 계열)로 검수하는 절차.** 이미지 생성기가 GPT(GPT-4o/gpt-image)이므로, **같은 계열 모델인 Codex**가 "이 프롬프트가 GPT 이미지 생성에 적절한지"를 판정하는 데 적합함. CLAUDE.md §7의 사실검수(환각 잡기)와 **다른** 검수임 — 여기선 **프롬프트 자체의 도메인 적합성**을 봄.

## 적용 범위
- **대상**: GPT 기반 이미지 프롬프트 — [[캐릭터 시트 프롬프트 템플릿]], [[스토리보드 프롬프트 템플릿]].
- **제외**: [[Seedance 모션 프롬프트 템플릿]] — Seedance는 ByteDance 모델이라 GPT 계열 검수의 도메인 적합 논리가 약함(영상 모델 특성은 별도 검증).

## 절차 (작성 → 검수 → 반영)
1. **작성** — 템플릿 슬롯을 채워 완성 프롬프트를 만듦.
2. **검수 (Codex)** — `codex:rescue` 스킬 또는 `codex` 계열 에이전트로 위임. 완성 프롬프트 + 용도(캐릭터 시트/스토리보드, 다운스트림=Seedance 첫 프레임)를 함께 전달함.
3. **반영** — Codex 판정을 보고 프롬프트를 수정. 판정 척도: **적절 / 경미한 수정 / 상당한 수정**.
4. **생성** — 통과한 프롬프트로 ChatGPT 이미지 생성.

## Codex 검수 체크리스트 (이미지 프롬프트용)
GPT 이미지 생성의 실제 동작 기준으로 점검함:

| 항목 | 점검 내용 |
|------|-----------|
| 크기/비율 | gpt-image-1 계열은 `1024x1024`·`1536x1024`·`1024x1536`만 지원. 16:9·2.39:1 등 미지원 비율을 요구하는지(→ gpt-image-2 또는 후처리 crop 필요) (출처: src-openai-api-imggen, src-openai-cookbook-genimg) |
| 객체 수 | 한 이미지 ~10~20개 한계. 한 장에 뷰/객체를 과하게 요구하는지 (출처: src-openai-cookbook-imggen) |
| 뷰 수 | 다중 뷰 시트를 한 장에 몰지 않고 분리했는지(전신 4뷰 + 헤드샷 등) |
| 일관성 장치 | 앵커 이미지·인덱스 참조·"Do not redesign"·기준 뷰 명시가 충분한지 (출처: src-openai-cookbook-imggen) |
| 스타일 충돌 | 강한 스타일 앵커가 캐릭터 정체성(얼굴·의상)을 덮지 않게 가드했는지 |
| 첫 프레임 품질 | (스토리보드) 모션블러·스피드라인·중간전이 포즈를 막아 영상 첫 프레임으로 안정적인지 |
| 모호성 | left/right 한 줄 묶기, 중복 스타일 슬롯 등 모델이 오해할 표현이 있는지 |
| 텍스트 라벨 | 이미지 내 텍스트 라벨 요구는 철자/위치 불안정 → 회피했는지 |

## 검수 호출 예
- 스킬: `codex:rescue` — "다음 이미지 프롬프트가 GPT-4o 이미지 생성에 적절한지 검수하라. [프롬프트 + 용도]".
- 에이전트: `codex:codex-rescue` 서브에이전트에 동일 요청 위임.
